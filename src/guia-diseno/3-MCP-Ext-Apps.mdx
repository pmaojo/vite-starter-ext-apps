# Integración con MCP Ext-Apps SDK

Bienvenido a la guía de integración del **Model Context Protocol (MCP)** mediante el SDK `@modelcontextprotocol/ext-apps`. Este documento está diseñado para enseñarte de forma detallada y didáctica cómo nuestras aplicaciones (Apps) conviven dentro de entornos conversacionales (como ChatGPT).

## ¿Qué es una App dentro de ChatGPT?

Las "Apps" son experiencias desarrolladas por terceros que extienden lo que los usuarios pueden hacer sin romper el flujo natural de la conversación. En lugar de llevar al usuario a una nueva pestaña del navegador, la App se presenta a través de interfaces ligeras y contextuales (como tarjetas, carruseles o vistas de pantalla completa) que se integran de manera fluida en la interfaz nativa.

El objetivo principal es: **Hacer al menos una cosa mejor gracias a que vivimos en la conversación.**

---

## Modos de Visualización (Display Modes)

Para asegurar que las experiencias se sientan nativas y predecibles, utilizamos "Modos de Visualización". Cada modo está diseñado para un tipo específico de interacción.

### 1. Inline (En línea)

Este es el modo predeterminado. El contenido aparece directamente en el flujo de la conversación, siempre antes de la respuesta generada por el modelo.

**Cuándo usarlo:**
- Decisiones rápidas o confirmaciones (ej. confirmar una reserva).
- Mostrar datos estructurados (ej. un resumen de orden o un mapa pequeño).
- Herramientas autocontenidas (ej. un reproductor de audio).

**Tipos de Componentes Inline:**
- **Inline Card (Tarjeta en línea):** Widgets de un solo propósito. Soportan interacción directa (ej. editar un texto rápidamente). *Regla de oro: Máximo dos botones (Call to Actions) y sin navegación profunda (sin pestañas o scroll interno).*
- **Inline Carousel (Carrusel en línea):** Una serie de tarjetas una al lado de la otra. Ideal para presentar entre 3 y 8 opciones comparables (ej. restaurantes, listas de reproducción).

### 2. Fullscreen (Pantalla Completa)

Son experiencias inmersivas que se expanden más allá de una tarjeta. Lo más importante aquí es que **el compositor de texto (el input de chat) de ChatGPT permanece visible**. Esto permite al usuario seguir "hablando con la app" mientras explora la vista.

**Cuándo usarlo:**
- Flujos de trabajo de múltiples pasos o tareas ricas (ej. editar un lienzo interactivo, un mapa explorable).
- Navegar por contenido muy detallado (ej. listados de bienes raíces).

*Regla de oro: Diseña pensando en el compositor de texto. La experiencia debe reaccionar a comandos de voz o texto del usuario.*

### 3. Picture-in-Picture (PiP)

Una ventana flotante y persistente optimizada para sesiones continuas (ej. juegos, colaboración en vivo). Se mantiene visible aunque el usuario haga scroll en la conversación y puede actualizarse dinámicamente con las respuestas del modelo.

## Principios de Experiencia de Usuario (UX)

Crear una gran aplicación no significa portar tu sitio web completo a un chat. Significa extraer el valor principal y adaptarlo al medio conversacional.

### 1. Extraer, no portar
Concéntrate en las tareas atómicas que tus usuarios necesitan. En lugar de un panel de control complejo (dashboard), proporciona herramientas claras que acepten las entradas (inputs) necesarias para que el modelo tome la siguiente acción de forma confiada.

### 2. Diseñar para entradas conversacionales
Los usuarios pueden llegar a tu App en medio de un chat, con intenciones claras ("Reserva la sala a las 3pm") o vagas ("Ayúdame a organizar un evento"). Tu interfaz debe soportar este contexto dinámico.

### 3. Optimizar para la conversación, no para la navegación
El modelo de IA maneja el estado y el enrutamiento. Tu App debe proporcionar:
- Acciones declarativas claras.
- Respuestas concisas (listas o tablas breves en lugar de muros de texto).
- Sugerencias útiles (follow-ups) para mantener al usuario en el flujo.

## Implementación Técnica con el SDK

En nuestro proyecto, utilizamos un componente central de React provisto por el SDK llamado `App`. Al renderizar nuestra aplicación de React, el hook `useApp()` nos proporciona la conexión con el "Host" (por ejemplo, ChatGPT).

### Arquitectura de Micro-Manifiestos

Nuestra base de código utiliza una arquitectura modular:
- Cada herramienta (`Tool`) vive en su propio directorio dentro de \`src/tools/\`.
- Expone un archivo \`manifest.ts\` que define su nombre (slug), título, descripción y el **Componente React** que lo renderiza.
- Cuando el Host decide invocar una herramienta, pasa un contexto (\`hostContext\`) que incluye el \`toolName\`. Nuestro enrutador principal en \`src/core/engine/mcp-app.tsx\` lee este nombre y renderiza dinámicamente el componente correcto.

Esto asegura que nuestra aplicación UI solo envíe al Host los componentes necesarios en el momento preciso, manteniendo la experiencia rápida y ligera, tal como lo dictan los principios UX del SDK.
