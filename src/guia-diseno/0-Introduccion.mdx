# Bienvenido a la Guía de Diseño Didáctica

Esta guía está especialmente diseñada tanto para **Diseñadores** como para **Desarrolladores**. El objetivo de este espacio no es solo documentar los componentes que tenemos, sino explicar **el porqué** de sus decisiones, **cómo** deben utilizarse y **cuándo** es apropiado usar cada variante.

## Para Diseñadores 🎨

Aquí encontrarás un reflejo directo de nuestro sistema de diseño. Cada componente está documentado con:
- **Estados:** Cómo se ve un componente cuando el usuario interactúa con él (hover, activo, deshabilitado).
- **Variantes:** Diferentes tipos de un mismo componente (ej. botón primario, secundario, destructivo).
- **Reglas de Uso:** Mejores prácticas de experiencia de usuario (UX) sobre cuándo usar o no un componente.

## Para Desarrolladores 💻

Además del aspecto visual, esta guía proporciona toda la información técnica necesaria para implementar los componentes:
- **Props y API:** Qué propiedades acepta cada componente, sus tipos y valores por defecto.
- **Ejemplos Prácticos:** Fragmentos de código listos para copiar y pegar en tu aplicación.
- **Accesibilidad (a11y):** Asegurarnos de que nuestros componentes cumplan con los estándares de accesibilidad desde su concepción.

## ¿Cómo navegar por esta guía?

En el panel lateral encontrarás varias secciones:
1. **Documentación Fundamental:** Como esta página, donde definimos colores, tipografía y espaciado.
2. **Componentes:** Los bloques de construcción de nuestra interfaz (Botones, Entradas de texto, Tarjetas).

¡Explora, interactúa con los componentes y utilízalos como base sólida para construir experiencias increíbles!
